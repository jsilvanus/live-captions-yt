# What this prompt does
Provides a structured, minimal prompt that captures the task goal, required
inputs, constraints, and desired output format so the assistant can act
consistently across similar developer tasks.

# Inputs (replace as needed)
- task: Short task title (one line)
- contextFiles: List of workspace paths that are relevant (optional)
- changeScope: Files or modules likely affected (optional)
- constraints: Any constraints (tests required, no API changes, follow style)
- preferredOutput: What to produce (patch, tests, explanation, commands)
- runTests: true/false — whether to run repo tests after changes

# Prompt template
You are a helpful, concise coding assistant.
Task: {{task}}
Context files: {{contextFiles}}
Change scope: {{changeScope}}
Constraints: {{constraints}}
Desired output: {{preferredOutput}}
Run tests: {{runTests}}

Act as a pair-programmer. Do the work in these steps:
1) Inspect the listed files (and the repo when needed) for minimal changes.
2) Propose a short plan of 2–4 steps and confirm assumptions briefly.
3) Apply changes using small, focused patches and run tests if requested.
4) Summarize what was changed, where, and next steps.

Output format:
- If code changes are required, return them via workspace edits (apply_patch).
- If tests are run, include results and failing traces.
- Finish with a short QA checklist (files changed, tests passed).

# Examples
Example 1 — Add unit tests for `src/foo.js`:
- task: "Add unit tests for foo.js"
- contextFiles: ["packages/lcyt/src/foo.js", "packages/lcyt/test/foo.test.js"]
- changeScope: "packages/lcyt/test"
- constraints: "use node:test, keep changes minimal"
- preferredOutput: "patch + run tests"
- runTests: true

Example 2 — Small CSS fix:
- task: "Fix Planner input clipping"
- contextFiles: ["packages/lcyt-web/src/styles/components.css"]
- changeScope: "packages/lcyt-web/src/styles"
- constraints: "preserve theme vars, avoid layout regressions"
- preferredOutput: "patch + brief verification steps"
- runTests: false

# Iteration tips
- If output is ambiguous, ask at most 2 clarifying questions.
- Prefer minimal, reversible edits and include tests where practical.
- For large or risky changes, propose a short plan and request confirmation.

# Where to save
- Workspace-level: place at repository root as `.prompt.md` or in `.github/prompts/`.
- Personal: copy to your editor prompts folder for reuse.
