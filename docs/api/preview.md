# /preview — Stream Thumbnail Preview

Serve a JPEG thumbnail of the incoming RTMP stream. Thumbnails are generated by the PreviewManager, which runs a continuous ffmpeg process per API key capturing one frame every 5 seconds.

**CORS:** `Access-Control-Allow-Origin: *` — open to all origins.

Rate-limited to **60 requests per minute per IP**.

---

## `GET /preview/:key/incoming.jpg` — Latest Stream Thumbnail

Return the most recently generated JPEG thumbnail for an incoming RTMP stream.

**Authentication:** None

**Request**

```http
GET /preview/my-api-key/incoming.jpg
```

**Response — `200 OK`**

```
Content-Type: image/jpeg
Cache-Control: public, max-age=5, must-revalidate
Last-Modified: <file mtime>
```

Conditional requests using `If-Modified-Since` are supported — the server returns `304 Not Modified` if the thumbnail has not been regenerated since the supplied timestamp. This avoids downloading the same image repeatedly during polling.

**Error responses**

| Status | Reason |
|---|---|
| `400` | Invalid key format |
| `304` | Thumbnail unchanged since `If-Modified-Since` |
| `404` | Preview not available (stream not live or ffmpeg not started) |
| `429` | Rate limit exceeded |

---

## Browser Polling Example

```html
<img id="preview" src="https://api.example.com/preview/my-api-key/incoming.jpg" alt="Live preview">
<script>
  const img = document.getElementById('preview');
  let lastModified = null;

  setInterval(async () => {
    const headers = lastModified ? { 'If-Modified-Since': lastModified } : {};
    const res = await fetch(img.src, { headers });
    if (res.status === 200) {
      lastModified = res.headers.get('Last-Modified');
      const blob = await res.blob();
      img.src = URL.createObjectURL(blob);
    }
    // 304 = no change, skip update
  }, 5000);
</script>
```

---

## Key Format

Preview keys must match `/^[a-zA-Z0-9_-]{3,}$/` (same rules as viewer and radio keys).

---

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `PREVIEW_ROOT` | `/data/preview` | Directory where JPEG thumbnails are stored. Each key gets its own file. |
