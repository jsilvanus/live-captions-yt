# /stream-hls — HLS Video Streaming

Endpoints for serving an HLS (HTTP Live Streaming) video+audio stream derived from the incoming RTMP signal. Also handles nginx-rtmp lifecycle callbacks.

The API key must have `hls_enabled = true` (set by an admin via `PATCH /keys/:key`).

---

## Public Streaming Endpoints

These endpoints are public (no authentication). CORS headers are set according to the per-key `embed_cors` setting (defaults to `*`).

All HLS file serving endpoints are **rate-limited to 120 requests per minute per IP**.

---

### `GET /stream-hls/:key/index.m3u8` — HLS Playlist

Return the live HLS playlist for the stream. The playlist is regenerated by ffmpeg roughly every 2 seconds.

**Request**

```http
GET /stream-hls/my-api-key/index.m3u8
```

**Response — `200 OK`**

```
Content-Type: application/vnd.apple.mpegurl
Cache-Control: no-cache, no-store
```

Returns the M3U8 playlist. Returns `404` if the stream is not currently live.

---

### `GET /stream-hls/:key/:segment` — HLS Segment

Fetch a specific video segment. Only segment filenames matching `seg00000.ts` through `seg99999.ts` are accepted.

**Request**

```http
GET /stream-hls/my-api-key/seg00001.ts
```

**Response — `200 OK`**

```
Content-Type: video/mp2t
Cache-Control: public, max-age=60
```

---

### `GET /stream-hls/:key/player.js` — Embeddable Video Player

Returns a self-contained vanilla-JavaScript HLS video player snippet. Include it as a `<script>` tag to embed the live stream in any web page.

**Request**

```http
GET /stream-hls/my-api-key/player.js
```

**Response — `200 OK`**

```
Content-Type: text/javascript; charset=utf-8
Cache-Control: public, max-age=3600
```

**Usage:**

```html
<!-- Optional: pre-create a container -->
<div id="hls-my-api-key"></div>

<!-- Load the player — self-contained, no dependencies required -->
<script src="https://api.example.com/stream-hls/my-api-key/player.js"></script>
```

The snippet creates a `<video>` element inside the container (or inserts one after the `<script>` tag). On browsers without native HLS support it loads `hls.js` from CDN automatically. On Safari and iOS (which support HLS natively), it sets `video.src` directly.

**Error responses**

| Status | Reason |
|---|---|
| `400` | Invalid key format or invalid path |
| `404` | Stream not live or segment not found |
| `429` | Rate limit exceeded |

## nginx-rtmp Callbacks

These endpoints are called by nginx-rtmp, not by browsers. They have no authentication — access should be restricted at the network level (nginx only).

### `POST /stream-hls` — Single-URL Style

```http
POST /stream-hls
Content-Type: application/x-www-form-urlencoded

call=publish&name=my-api-key
```

or

```
call=publish_done&name=my-api-key
```

### `POST /stream-hls/on_publish`

```http
POST /stream-hls/on_publish
Content-Type: application/x-www-form-urlencoded

name=my-api-key
```

### `POST /stream-hls/on_publish_done`

```http
POST /stream-hls/on_publish_done
Content-Type: application/x-www-form-urlencoded

name=my-api-key
```

**Behavior:**
- `publish` — starts an ffmpeg process that transcodes the incoming RTMP stream to HLS under `$HLS_ROOT/<key>/`. Returns `403` if `hls_enabled` is not set for the key.
- `publish_done` — stops the ffmpeg HLS process for the key.

All nginx callbacks return plain text `ok` on success, or an error message with the appropriate HTTP status.

---

## nginx-rtmp Configuration Example

```nginx
application hls {
  live on;

  # Single-URL style
  on_publish      http://127.0.0.1:3000/stream-hls;
  on_publish_done http://127.0.0.1:3000/stream-hls;

  # Or separate-URL style
  # on_publish      http://127.0.0.1:3000/stream-hls/on_publish;
  # on_publish_done http://127.0.0.1:3000/stream-hls/on_publish_done;
}
```

## Environment Variables

| Variable | Default | Description |
|---|---|---|
| `HLS_ROOT` | `/data/hls` | Directory where HLS playlists and segments are written. Each key gets its own subdirectory. |
| `BACKEND_URL` | _(derived from request)_ | Used to build absolute URLs in `player.js`. Set this when the backend is behind a reverse proxy. |
